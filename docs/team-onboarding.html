1. 프로젝트 개요

Match-up은 테니스·풋살 동호인이 겪는 4가지 페인 포인트 (종목별 일반 규칙 · 대회별 규칙 · 본인 등급 출전 가능 대회 · 최신 대회 일정)을 한 앱에서 해결합니다.

핵심 가치 — 회원가입 시 종목·등급을 등록하면 본인 등급으로 출전 가능한 대회만 자동 필터링됩니다. tournaments_for_user RPC가 종목별 매칭을 처리.

스택 한눈에

영역선택
FrontendFlutter (Riverpod + go_router) — iOS / Android / 웹 / macOS
BackendSupabase Edge Functions (Deno) — 모든 서버 로직 일원화
DBPostgres + pgvector + pg_cron + pg_net
AIGemini 2.0 Flash + Search Grounding · gemini-embedding-001 (768d)
AuthSupabase Auth (이메일 + 구글, 카카오는 v2)
PushFCM

종목·등급 모델

종목enum표시
tennisrookie div5 div4 div3 div2 div1신입 / 5부 / 4부 / 3부 / 2부 / 1부
futsalbeginner intermediate advanced초급 / 중급 / 고급

한 사용자가 두 종목 모두 등록 가능 (user_sports N:M).

2Docker Desktop

이미 설치되어 있으면 건너뛰세요. 확인:

docker --version
docker info --format '{{.ServerVersion}}'   # server 버전 출력되면 daemon 동작 중

설치:

# Apple Silicon
open https://desktop.docker.com/mac/main/arm64/Docker.dmg

# Intel
open https://desktop.docker.com/mac/main/amd64/Docker.dmg

설치 후 Docker.app을 한 번 실행해서 daemon을 켜야 합니다 (Launchpad → Docker).

⚠️ Apple Silicon 사용자는 Rosetta 활성화 옵션 OFF 권장. Docker → Settings → General에서 "Use Rosetta for x86_64/amd64 emulation" 체크하면 일부 이미지 호환성이 늘지만 지금은 불필요.

# 3) 첫 실행 컴포넌트 (시뮬레이터 런타임 등) 자동 설치

# 검증

⚠️ 여기까지 안 하면 macOS desktop 빌드가 실패합니다 — xcodebuild 또는 CocoaPods 누락 에러 발생.

7Gemini API 키 발급 — 본인 키 필수

🔒 키는 절대 공유받지 마세요. 사용자(레포 owner)도 본인 개인 키를 쓰고 있고, 같은 키를 공유하면 비용·할당량이 합쳐지고 노출 시 한 명 회전이 다른 사람 영향을 미칩니다. 무료 한도 안에서 본인 키로 충분히 개발 가능합니다.
  1. Google AI Studio 접속 (본인 Google 계정으로)
  2. "Create API key" 클릭 → 새 프로젝트 또는 기존 GCP 프로젝트 선택
  3. 발급된 키 (AIzaSy...로 시작) 복사 — 이 키는 본인만 사용
💡 무료 한도 충분 — Gemini 2.0 Flash + embedding은 일일 무료 한도 안에서 개발에 부족함 없음. 실 운영 배포 시에만 결제 활성화 필요. (운영 배포는 SSF-277에서 별도)

레포의 supabase/functions/ 디렉토리에 .env 파일 생성:

# Match-up 디렉토리에서
mkdir -p supabase/functions
cat > supabase/functions/.env <<EOF
GEMINI_API_KEY=AIzaSy...본인키
GEMINI_MODEL=gemini-2.0-flash
GEMINI_EMBEDDING_MODEL=gemini-embedding-001
EOF
🔒 이 파일은 .gitignore로 차단되어 있습니다 — git에 commit되지 않습니다. 반드시 본인 키만 사용.

8Supabase 로컬 스택 기동 — 본인 머신

🏠 Supabase 클라우드 프로젝트는 따로 만들 필요 없습니다. 본인 macOS에 로컬 스택을 띄우는 것만으로 개발 OK. 12개 컨테이너(Postgres + Auth + Edge Runtime + Storage + Studio + Kong 등)가 docker로 돕니다.

Docker Desktop이 실행 중인 상태에서 첫 실행은 ~5분 (이미지 ~2GB 다운로드):

cd ~/Documents/Match-up

supabase start              # 또는 supabase-beta start

성공하면 다음과 비슷한 출력:

✓ Started supabase local development setup.

API URL    : http://127.0.0.1:54321
DB URL     : postgresql://postgres:postgres@127.0.0.1:54322/postgres
Studio URL : http://127.0.0.1:54323
Publishable: sb_publishable_XXXXXXXXXXXXXXXXXXXXXXXX

Publishable 키는 메모해두세요 (Flutter 실행 시 필요).

마이그레이션 + 시드 적용:

supabase db reset

이후 시드 13건 (룰북 8 + 클럽 5 + 대회 5) + 마이그레이션 8개 + RPC 8개가 적용됩니다.

스택 검증 SQL 
# 테이블 10개 확인
docker exec -e PGPASSWORD=postgres supabase_db_matchup \
  psql -U postgres -d postgres \
  -c "select tablename from pg_tables where schemaname='public' order by tablename;"

# 시드 카운트
docker exec -e PGPASSWORD=postgres supabase_db_matchup \
  psql -U postgres -d postgres \
  -c "select 'tournaments', count(*) from public.tournaments
       union all select 'rule_articles', count(*) from public.rule_articles
       union all select 'clubs', count(*) from public.clubs;"

11E2E 시나리오 (앱에서 클릭)

단계화면동작
A로그인"계정이 없어요. 가입하기" → 이메일 + 비번 (6자+) → 회원가입
B온보딩테니스 카드에서 "3부" 칩 + 우상단 "주 종목" 라디오 ON / 풋살 카드에서 "중급" 칩 → 시작하기
C홈3건 표시: 광주광역시장배 + 풋살 챌린지컵 + 풋살 입문
D종목 토글전체(3) / 테니스(1) / 풋살(2) 칩 전환
E즐겨찾기카드 우측 북마크 → ON/OFF
F대회 상세카드 탭 → 등급/날짜/장소/참가비 표시
G챗봇"광주에서 열리는 테니스 대회 알려줘" → SSE 스트리밍 + DB 인용 ("광주광역시장배") + 웹 검색 인용
H챗봇"테니스 발리 시 라인 안에 있어도 되나요?" → 룰북 인용 (rule_articles)

12. 트러블슈팅 (자주 막히는 곳)

🔴 가입 시 "Connection failed (errno = 1, Operation not permitted)"

원인: macOS App Sandbox가 외부 네트워크를 막음.

해결: app/macos/Runner/DebugProfile.entitlements와 Release.entitlements 둘 다에 다음 추가:

<key>com.apple.security.network.client</key>
<true/>

그 후 앱 종료 후 flutter run -d macos ... 재실행.

이 프로젝트는 이미 적용되어 있지만 flutter create로 macos 폴더를 새로 만든 경우 다시 추가해야 함.

🔴 "Error: CocoaPods not installed or not in valid state"

brew install cocoapods + pod --version 로 검증.

🔴 "xcrun: error: unable to find utility 'xcodebuild'"

Xcode CLT만 깔린 상태. Xcode 본체 설치 + sudo xcode-select -s /Applications/Xcode.app/Contents/Developer.

🔴 supabase start 실패: "port 54321 already allocated"

이전 세션의 컨테이너가 살아있음. 정리:

docker ps -a --filter name=supabase_ -q | xargs -r docker rm -f
supabase start
🔴 챗봇 "[연결 실패]" 또는 401

가능 원인:

  1. functions serve가 안 떠 있거나 .env 로드 안 됨 → 재시작
  2. GEMINI_API_KEY가 invalid → AI Studio에서 새 키 발급 후 .env 교체 + functions serve 재시작
  3. 로그인 토큰 만료 → 앱 종료 후 재로그인
🟡 가입 버튼 눌렀는데 반응 없음

대부분 가입은 됐지만 자동 로그인이 안 된 상태. flutter run 터미널에서 R (대문자, hot restart) → 저장된 세션 복구로 자동 로그인 → 온보딩 화면.

그래도 안 되면 "이미 계정이 있어요" 클릭하고 로그인 모드로 시도.

🟡 홈 화면이 비어 있음 (대회 0건)

가능 원인:

  1. 온보딩에서 등급 등록이 안 됨 → 프로필 화면 → "수정"으로 다시 등록
  2. 시드가 적용 안 됨 → supabase db reset 다시
  3. "내 등급만" 토글이 ON인데 매칭 대회 0건 → 토글 OFF 또는 다른 등급으로 변경
🟡 Flutter web (Chrome)에서 빌드 실패

알려진 이슈 (SSF-283): app/lib/services/api.dart가 dart:io를 사용 → web에서 빌드 깨짐.

해결 전까지 macOS desktop 또는 iOS Simulator 사용.

🟡 Docker daemon 안 떠 있음

open -a Docker 실행 후 ~30초 대기. 메뉴바 고래 아이콘이 멈추면 ready.
