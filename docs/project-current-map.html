제품 스펙

제품 정의

Match-up은 사용자가 종목과 등급을 등록하면 출전 가능한 대회를 자동으로 보여주고, 대회 정보·클럽·룰북·AI 챗봇·푸시 알림·스피드건을 연결하는 동호인 정보 앱이다.

tennis futsal grade-aware search RAG chatbot

기술 스택

  • Frontend: Flutter/Dart, Riverpod, go_router, Material 기반 커스텀 디자인 시스템
  • Backend: Supabase Edge Functions on Deno
  • DB: Supabase Postgres, RLS, RPC, pgvector, pg_cron, pg_net, Vault
  • AI: Gemini, Search Grounding, 768차원 임베딩 기반 RAG
  • Push: Firebase Cloud Messaging

현재 구현된 주요 기능

영역 구현 상태 작동 방식
인증·온보딩 이메일/비밀번호, Google OAuth, 개발용 Dev 퀵로그인, 종목·등급 등록 로그인 후 user_sports가 없으면 /onboarding으로 이동한다. 개발 모드에서는 dev-auth가 magic link 토큰을 만들어 빠르게 로그인한다.
종목·등급 테니스와 풋살 모두 지원. 테니스는 협회별 다중 등록 구조까지 있음 user_sports와 user_tennis_orgs를 기준으로 active sport와 등급을 정하고, 대회 검색 RPC가 종목 교차 매칭을 막는다.
대회 검색 등급 기반 검색, 지역/기간/검색어 필터, 즐겨찾기 Flutter가 tournaments-search를 호출하고, 서버/DB가 사용자의 등급과 eligible_grades를 비교해 노출 대회를 결정한다.
대회 제보·검수 사용자 제보는 draft, 어드민 승인 후 published, 거부는 rejected 어드민 화면의 검수 큐에서 사용자 제보와 크롤러 draft를 필터링하고, 일괄 승인/거부 RPC로 상태를 바꾼다.
크롤러 DB-driven crawl-dispatch, 크롤 소스 CRUD, 수동 실행, 중복 실행 방지 pg_cron이 crawl-dispatch를 호출하고, dispatcher가 crawl_sources의 enabled source를 parser_module에 매핑해 실행한다.
클럽 클럽 생성 요청, 어드민 승인, 내 클럽, 가입 신청, 가입 승인/거절, 탈퇴 클럽은 pending → approved/rejected 흐름을 타고, 멤버십은 club_members, 가입 요청은 club_join_requests로 관리한다.
클럽 활동 부분 구현. DB 마이그레이션, 모델, API 메서드, 상세 화면 초안 존재 035_club_events.sql, club_event.dart, ApiService.clubEvents/createClubEvent/respondEvent, ClubDetailScreen, /clubs/:id 라우트가 있다. 다만 기존 클럽 목록은 아직 바텀시트 상세를 열고 있어 진입 연결 마무리가 필요하다.
AI 챗봇 SSE 스트리밍, RAG, Search Grounding, 대화 이력, 시맨틱 검색 사용자 질문을 임베딩하고 DB 문서·대회 컨텍스트를 검색한 뒤 Gemini로 답한다. 응답은 SSE로 조각 단위 수신된다.
지식베이스 어드민 룰북 CRUD, 게시 여부, 임베딩 상태, 즉시 재계산 어드민이 rule_articles를 직접 관리하고, 저장 시 임베딩을 무효화한 뒤 embed-pending으로 재계산할 수 있다.
푸시 알림 디바이스 토큰 등록, D-3/마감 알림 워커 구조 로그인 후 FCM 토큰을 device_tokens에 저장하고, notify-cron이 중복 방지 로그를 기준으로 발송한다.
스피드건 비디오 선택, 코트 캘리브레이션, 속도 계산 서비스, 결과 화면 구조 240fps 영상에서 프레임을 추출하고, 공 위치·코트 보정·속도 계산을 연결한다. 모델 파일이 없으면 개발용 mock detector가 동작한다.
디자인 시스템 Pretendard, 라이트/다크 테마, 공통 위젯, Figma Make 반영 app/lib/theme와 app/lib/widgets의 토큰/컴포넌트로 화면 스타일을 통일하고 있다.
보안·운영 RLS, admin role, cron JWT Vault 전환, secret scan, CI/harness cron 호출은 Vault의 internal_cron_jwt를 사용하도록 개선됐다. 다만 legacy API key 완전 전환과 노출 JWT revoke는 남은 작업이다.

현재 리포 기준 주의할 점

미커밋 변경

supabase/migrations/035_club_events.sql, app/lib/models/club_event.dart, app/lib/screens/clubs/club_detail_screen.dart, 그리고 app/lib/services/api.dart의 클럽 활동 메서드가 아직 커밋되지 않은 변경이다. 클럽 활동 MVP의 일부 구현으로 보이며, 팀 작업 전에는 리뷰와 테스트가 필요하다.

문서와 실제 코드 차이

일부 계획 문서에는 crawl dispatcher가 대기 상태로 남아 있지만, 실제 코드에는 crawl-dispatch와 cron 전환 마이그레이션이 존재한다. 팀 공유 시에는 이 HTML처럼 실제 코드 기준으로 설명하는 편이 안전하다.
